Help & Documentation

Data Dictionary

bank_transactions: Core transaction records from bank feeds. Fields: transactionDate, amount, direction, counterparty, description, reference.
categories: Hierarchical expense categories. Level 2 (e.g., INVEST_CONSTRUCTION) and Level 3 (e.g., GROS OEUVRES).
suppliers: Vendor/counterparty directory with default category assignments.
budgets: Planned amounts by category and/or supplier.

KPI Definitions

Total CAPEX: Sum of all outflows excluding internal transfers and bank fees, filtered to INVEST_* categories.
Budget Remaining: Total budgeted amount minus actual spend to date.
Burn Rate: Average monthly spend over the selected rolling period (30/60/90 days).
Runway: Current cash position divided by monthly burn rate.
Funding Coverage: Total funding received divided by total spend to date.
Supplier Concentration: Share of total spend held by top 5 suppliers.

Auto-Categorization Rules

The system automatically categorizes transactions using this priority order:

  1. Manual override (highest priority)
  2. Exact match rules on counterparty
  3. Contains match rules
  4. Regex pattern rules
  5. Supplier default category
  6. Mark as "needs review" if unmatched
